# Introduction

`tsontology` is best understood as a **pre-model audit and communication layer** for time-series work.

Instead of starting from “which model should I train?”, it starts from:

- what kind of temporal structure is in this dataset?
- what does that usually imply for modelling and validation?
- how can I explain the data to another human or another agent?

That is why the package now emphasizes report generation, summary cards, HTML pages, and compact tool outputs.
